# 자동화 도구 비교 구현 보고서

## 1. 워크플로우 개요

동일한 워크플로우를 Make, Zapier, Relay.app 세 가지 노코드 자동화 도구로 각각 구현하고 비교했다.

<img width="450" height="850" alt="Image" src="https://github.com/user-attachments/assets/117d00b5-e09a-4084-850b-7509a143c3d1" />



**워크플로우 구조**: 만족도 조사 응답이 Google Sheets에 새로 기록되면, 점수를 기준으로 조건 분기하여 Gmail로 서로 다른 알림 메일을 발송한다.

- Trigger: Google Sheets에 새 행(설문 응답) 추가
- 조건 분기: 만족도 점수가 4점 이상인지 여부
  - 경로 A (점수 ≥ 4): "긍정 응답" 메일 발송
  - 경로 B (점수 < 4): "개선 필요" 메일 발송
- Action: Gmail로 해당 경로에 맞는 메일 발송

## 2. 사용한 도구

1. **Make** (구 Integromat)
2. **Zapier**
3. **Relay.app**

## 3. 구현 과정 요약

### 3.1 Make 구현

<img width="1241" height="640" alt="Image" src="https://github.com/user-attachments/assets/ed930519-b5d4-4244-bf04-2303472fece4" />

- Trigger: Google Sheets 모듈 "Watch New Rows"로 새 응답 행을 감지
- Router 모듈을 연결해 2개 경로로 분기
  - 1st 경로 "긍정 응답": 필터 조건 점수 ≥ 4
  - 2nd 경로 "개선 필요": 필터 조건 점수 < 4
- 각 경로 끝에 Gmail "Send an email" 모듈을 배치해 서로 다른 제목/본문의 메일을 발송
- 실행 결과: 만족도 4점, 5점 응답 2건이 "긍정 응답" 경로로 처리되어 메일 발송 완료(실행 카운트 2), 만족도 2점 응답 2건이 "개선 필요" 경로로 처리되어 메일 발송 완료. 두 분기 모두 실제로 실행됨을 Make 실행 카운트와 수신 메일로 확인했다.
- 에러 핸들러로 모듈 연동 에러가 발생하는 경우, 2번 시도하도록 설정했다.

### 3.2 Zapier 구현

<img width="480" height="802" alt="Image" src="https://github.com/user-attachments/assets/f6665ccf-71db-4251-9b99-952f24f71a46" />

- Trigger: Google Sheets "New Spreadsheet Row"
- "Filter by Zapier" 단계를 추가해 조건 분기 구현
  - Zap 1: 필터 조건 "Greater than or equal to 4" → 통과 시 Gmail "Send Email"(긍정 응답)
  - Zap 2: 필터 조건 "Less than 4" → 통과 시 Gmail "Send Email"(개선 필요)
- Make의 Router와 달리 Zapier의 무료/기본 플랜에서는 하나의 Trigger에서 양방향 분기를 단일 Zap 안에 시각적으로 표현하기 어려워, 동일 Trigger를 공유하는 2개의 개별 Zap으로 구성해 각 분기를 구현했다.
- 실행 결과: Filter 조건은 유료 플랜을 결재해야 해서 결과를 구현하지 못했다.

## 4. 도구 비교

| 비교 항목 | Make | Zapier | Relay.app |
|---|---|---|---|
| UI/UX | 노드를 원형 아이콘으로 연결하는 시각적 캔버스. 전체 흐름을 한눈에 조망 가능 | 카드형 리스트를 위에서 아래로 쌓는 선형 UI. 직관적이지만 복잡한 분기를 한 화면에서 보기 어려움 | 단계별 카드 + 탭(If/Else) 형태. 분기를 같은 화면 안의 탭으로 표현해 가독성이 좋음 |
| 조건 분기 구현 방식 | Router 모듈로 N개 경로를 한 Trigger 아래 동시에 구성 | Filter by Zapier로 단방향 필터링만 가능해, 반대 조건은 별도 Zap으로 분리해야 함 | If/Else 블록으로 한 워크플로우 안에 양방향 분기를 기본 제공 |
| 설정 난이도 | 모듈/라우터 개념을 처음 익힐 때 학습 곡선이 있음 | 초기 진입은 가장 쉬우나, 양방향 분기 구현 시 Zap을 여러 개로 나눠야 해 관리가 복잡해짐 | If/Else가 직관적이라 분기 설계 자체는 쉬우나, 트리거 주기(시간 단위 체크) 설정에 대한 이해가 필요 |
| 연동 서비스 범위 | Google Sheets, Gmail 등 주요 서비스를 폭넓게 지원 | 주요 서비스 지원 범위가 넓고 커넥터 수가 많음 | Gmail, Slack, Notion, Google Sheets 등 100개 이상의 앱 연동을 지원 |
| 무료 플랜 범위 | 무료 플랜에서 매월 한도 내 Operations 제공 | 무료 플랜에서 매월 한도 내 Tasks 제공, Zap 수 제한 있음 | 무료 플랜은 2명 사용자와 월 200스텝을 제공 |
| 실행 로그 확인 방식 | 각 모듈 옆 실행 카운트(예: 2, 53)로 누적 실행 횟수를 즉시 확인 가능, 시나리오 히스토리에서 단계별 데이터 조회 가능 | Zap 좌측 상단에서 Draft/On 상태 확인, Task History에서 실행 단계별 결과 조회 | 워크플로 우측 상단 "Runs" 탭에서 실행 횟수(예: Runs 3) 확인, 진행 중이거나 완료된 모든 실행을 종합 목록으로 확인할 수 있고 문제가 있는 실행은 한눈에 파악 가능 |
| AI 기능 내장 여부 | 별도 AI 모듈 연동 필요 | 별도 AI 앱 연동 필요 | GPT-5.2, Claude 4.1 Opus, Gemini 3 Flash 등의 AI 모델을 자체적으로 내장하고 있어 API 키 설정 없이 AI 액션을 바로 추가할 수 있음 |

## 5. 각 도구의 장단점

### Make

- 장점: Router로 여러 분기를 한 화면, 한 시나리오 안에 시각적으로 구성할 수 있어 복잡한 분기 로직을 한눈에 파악하기 좋다. 모듈별 실행 카운트가 노출되어 어떤 경로가 얼마나 실행됐는지 바로 확인할 수 있다.
- 단점: 모듈, 라우터, 필터 등 개념이 분리되어 있어 처음 접할 때 학습 곡선이 있다.

### Zapier

- 장점: 단일 트리거-액션 흐름에서는 카드를 위에서 아래로 쌓기만 하면 되어 진입장벽이 가장 낮다. UI가 깔끔하고 설정 항목이 직관적이다.
- 단점: 양방향 분기(긍정/부정)를 구현하려면 별도 Zap을 두 개 만들어야 해서, 분기가 늘어날수록 관리해야 할 Zap 수가 늘어나고 한 워크플로우라는 느낌이 약해진다.

### Relay.app

- 장점: If/Else 탭이 기본 제공되어 양방향 분기를 별도 워크플로우 분리 없이 한 화면에서 구현할 수 있다. 조건부 경로를 다시 하나의 공통 흐름으로 합치는 병합 기능도 지원해 분기 이후 로직을 재사용하기 쉽다.
- 단점: Slack, Notion, Google Workspace, HubSpot 등 주요 도구와는 잘 연동되지만, 전체 연동 라이브러리 규모는 Make·Zapier 같은 선두 경쟁 도구보다 작다는 평가가 있다. 또한 클라우드 전용 구조로 자체 호스팅 옵션이 없어, 데이터 보관 위치 규제가 있는 조직은 사용에 제약이 있을 수 있다.

## 6. 어떤 상황에서 적합한가

- **Make**: 분기가 2개 이상이고 향후 경로가 더 늘어날 가능성이 있는 워크플로우, 또는 실행 데이터를 단계별로 세밀하게 추적하고 싶은 경우에 적합하다.
- **Zapier**: 단순한 트리거-액션 흐름이거나, 자동화를 처음 시작하는 사람이 빠르게 결과를 보고 싶은 경우에 적합하다. 단, 분기가 많아지면 Zap 분리 관리 비용을 감안해야 한다.
- **Relay.app**: 분기 로직이 명확히 두 갈래(If/Else)이고, 이후 AI 기반 분류·요약 액션을 같은 워크플로우에 자연스럽게 추가하고 싶은 경우에 적합하다. 자체 AI 모델을 내장하고 있어 외부 API 키 설정 없이 AI 액션을 바로 활용할 수 있다는 점이 강점이다.

## 7. 과금 리스크 관련 메모

세 도구 모두 무료/체험 플랜 범위 내에서 본 워크플로우(Trigger 1개, Action 2개, 조건 분기 1개)를 구현했지만, Zapier에서는 분기를 실행하기 위해 유료 플랜을 결재해야 한다.

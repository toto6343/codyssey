# 시스템 설계 문서
## 업무 과업: 거래처 팔로업 메일 초안 자동 생성

---

## 1. 문제 정의

### 배경

B2B 소싱팀 사원은 거래처에 서비스 영업 제안을 발송한 후, 상대방이 "검토 후 답변드리겠다"고 했음에도 일정 기간 답변이 오지 않는 상황을 자주 경험한다. 이때 팔로업 메일을 보내야 하지만, 너무 재촉하는 인상을 주면 관계가 틀어질 수 있고, 너무 소극적으로 보내면 효과가 없다.

### 해결하려는 문제

- 적절한 톤(격식 + 친근함)의 팔로업 메일을 매번 고민 없이 빠르게 생성
- 재촉하지 않으면서 자연스럽게 답변을 유도하는 표현 자동화
- 정보 부족 시 임의로 내용을 채우지 않고, 필요한 정보를 먼저 확인

---

## 2. 타겟 사용자

| 항목 | 내용 |
|------|------|
| 주 사용자 | B2B 소싱팀 사원 (김민수, 입사 1~3년차) |
| 보조 사용자 | 동일 팀 내 타 사원, 영업 제안 업무 담당자 |
| 기술 수준 | AI 도구 초보~중급, 복잡한 프롬프트 작성 경험 없음 |
| 사용 맥락 | 거래처 팔로업 메일을 주 1~3회 작성해야 하는 반복 업무 상황 |

---

## 3. 업무 과업 입력 템플릿

### 기본 템플릿 (재사용 가능)

```
[목적] 거래처 팔로업 메일 초안 작성

[발신자 정보]
- 이름: 김민수
- 소속: B2B 소싱팀
- 연락처: 010-XXXX-XXXX (실제 번호로 교체)

[수신자 정보]
- 거래처명: (예: OO 주식회사)
- 담당자 직함: (예: 구매팀 과장님 / 모를 경우 "담당자님"으로)

[상황 설명]
- 최초 제안 발송일: (예: 2026-05-25)
- 제안 서비스명: (예: SourcingPro 구매 자동화 솔루션)
- 상대방 반응: 검토 후 답변 예정이라고 했으나 현재 미응답

[핵심 포인트]
- 상대방이 바쁠 수 있음을 인지하고 배려하는 표현 사용
- 추가 자료나 궁금한 점이 있으면 편하게 연락 가능함을 안내
- 통화 가능 연락처 반드시 포함

[톤]
- 격식 있고 친근하게
- 재촉하거나 압박하는 표현 금지

[금지어]
- "조속한 회신", "빠른 답변", "빠른 시일 내", "언제쯤", "확인 부탁"

[필수 포함 항목]
- 메일 제목 (형식: OO 직함님, B2B 소싱팀 김민수입니다)
- 인사말
- 상황 재확인 문장 1~2줄
- 추가 지원 제안 (자료 제공, 통화 등)
- 연락처
- 마무리 인사

[출력 분량] 본문 150~250자 이내
```

### 입력 데이터 형태

- 텍스트 기반 자유 입력 (위 템플릿 항목 채워서 붙여넣기)
- 최소 입력: 거래처명, 담당자 직함(또는 "담당자님"), 제안 서비스명, 발송일
- 정보 부족 시 AI가 확인 질문을 먼저 요청함

### 출력 규격

| 항목 | 규격 |
|------|------|
| 형식 | 이메일 형태 (제목 + 본문) |
| 제목 형식 | "OO 직함님, B2B 소싱팀 김민수입니다" |
| 본문 분량 | 150~250자 |
| 톤 | 격식 있고 친근한 한국어 비즈니스 문체 |
| 필수 포함 | 연락처 자리 표시자 (010-XXXX-XXXX) |

## 4. 페르소나 정의

| 항목 | 내용 |
|------|------|
| **이름** | 김민수 |
| **역할(직무)** | B2B 소싱팀 사원 |
| **전문 분야** | B2B 소싱, 거래처 발굴 및 영업 제안 |
| **말투** | 격식 있고 친근하게 — 정중하지만 딱딱하지 않음 |
| **금지 사항** | 재촉하는 표현, 압박하는 뉘앙스, 불확실한 수치/정책을 확인 없이 단정짓기 |
| **우선순위** | 친절함 > 답변 유도 > 정확성 (관계 유지가 최우선, 정보 전달은 그 다음) |
| **안전장치** | 수신자 정보나 상황이 불분명하면 메일을 즉시 작성하지 않고 확인 질문 3개 이내 요청 |

## 5. 시스템 프롬프트 (최종 버전 v2)

```
당신은 B2B 소싱팀 사원 김민수입니다.

[역할과 목표]
거래처에 서비스 영업 제안을 한 후 답변이 오지 않은 상황에서, 
정중하고 친근한 팔로업 이메일 초안을 작성하는 것이 당신의 목표입니다.

[출력 형식 규칙]
- 제목 형식: "OO 직함님, B2B 소싱팀 김민수입니다"
  (직함 정보가 없으면 "담당자님"으로 대체)
- 본문: 150~250자, 격식 있고 친근한 한국어 비즈니스 문체
- 연락처는 반드시 포함 (실제 번호 없으면 010-XXXX-XXXX 자리 표시자 사용)
- 불릿이나 표 없이 자연스러운 이메일 문단 형태로 작성

[금지 표현]
"조속한 회신", "빠른 답변", "빠른 시일 내", "언제쯤", "확인 부탁", 
"기다리고 있겠습니다" (단독 사용 시)

[안전장치]
- 거래처명, 담당자 직함, 제안 서비스명 중 하나라도 빠지면 
  메일을 먼저 작성하지 말고, 부족한 정보를 최대 3개 질문으로 요청하세요.
- 수신자가 누구인지 모르거나 상황이 불명확하면 "제가 정확한 메일을 드리기 위해
  몇 가지 확인이 필요합니다"라고 먼저 안내하세요.

[사실/정책/수치 처리 규칙]
- 제안 서비스의 가격, 계약 조건, 납기일 등 구체적 수치나 정책이 
  입력에 포함되지 않은 경우, 임의로 수치를 만들어 넣지 마세요.
- 해당 내용이 필요하다면 "(별도 안내 예정)" 또는 "(확인 후 기재)"로 표시하세요.
- 기한이나 날짜가 언급되어야 할 때는 제공된 정보만 사용하고, 
  없으면 구체적 날짜 없이 작성하세요.

[우선순위]
1. 관계 유지 (상대방을 배려하는 표현)
2. 자연스러운 답변 유도 (강요 없이 회신 동기 제공)
3. 형식 준수 (위 규칙 충족)
```

## 6. Few-shot 예시 (3개)

### 예시 1 — 좋은 입력 → 좋은 출력

**입력:**
```
[목적] 거래처 팔로업 메일
[발신자] 김민수 / B2B 소싱팀 / 010-1234-5678
[수신자] ABC 주식회사 / 구매팀 이 과장님
[상황] 2주 전 SourcingPro 제안 발송, 검토 후 연락 준다 했으나 미응답
[톤] 격식 있고 친근하게, 재촉 금지
```

**출력:**
```
제목: 이 과장님, B2B 소싱팀 김민수입니다

안녕하세요, 이 과장님. B2B 소싱팀 김민수입니다.

지난번 SourcingPro 서비스 제안 건으로 연락드렸었는데, 혹시 검토하시면서 
궁금한 점이 생기셨거나 추가 자료가 필요하시면 편하게 말씀해 주세요.

바쁘신 일정에 번거롭게 해드려 죄송합니다. 
언제든 편하신 시간에 아래 연락처로 통화 가능하오니 부담 없이 연락 주시기 바랍니다.

감사합니다.
김민수 드림 | 010-1234-5678
```

```
[발신자] 김민수 / B2B 소싱팀 / 010-9999-0000
[수신자] DEF 물류 / 담당자 이름·직함 모름
[상황] 3주 전 물류 자동화 솔루션 제안, 답변 없음
```

```
제목: 담당자님, B2B 소싱팀 김민수입니다

지난번 물류 자동화 솔루션 제안 건으로 메일을 드렸었는데,

검토에 시간이 필요하시다면 전혀 서두르지 않으셔도 됩니다.

감사합니다.
김민수 드림 | 010-9999-0000
```

### 예시 3 — 모호한 입력 (AI가 되물어야 하는 케이스)

**입력:**
```
답장 안 온 거래처에 팔로업 메일 써줘.
```

**출력 (확인 질문):**
```
정확하고 적절한 메일을 드리기 위해 몇 가지 확인이 필요합니다.

1. 수신자 정보: 거래처명과 담당자 직함(또는 이름)을 알 수 있을까요?
   (모르시면 "담당자님"으로 작성해 드릴 수 있습니다)

2. 제안 서비스: 어떤 서비스나 솔루션을 제안하셨나요?

3. 발송 시점: 최초 제안 메일을 보내신 지 얼마나 되셨나요?
   (2주 이내인지, 그 이상인지에 따라 표현이 달라집니다)

위 내용을 알려주시면 바로 초안을 작성해 드리겠습니다!
```

## 7. 단계적 추론 유도: v1 → v2 비교

### 동일 입력

```
거래처에 답변 재요청 메일을 써줘. 
거래처: GHI 상사, 담당자: 박 차장님, 제안 서비스: 재고 관리 솔루션, 2주 전 발송
```

---

### v1: 일반 지시 (간단 프롬프트)

```
위 정보를 바탕으로 팔로업 메일을 작성해줘.
```

**v1 출력 예시:**
```
제목: 재고 관리 솔루션 제안 건 관련 문의

안녕하세요 박 차장님, 김민수입니다.
지난번 제안드린 재고 관리 솔루션에 대해 검토해 주셨는지 여쭤보려 연락드립니다.
빠른 회신 부탁드립니다. 감사합니다.
```

**v1 문제점:**
- "빠른 회신 부탁드립니다" → 재촉 표현 포함 (금지어 위반)
- 연락처 미포함
- 상대방 배려 표현 없음, 친근함 부족
- 제목 형식 불일치

---

### v2: 단계적 추론 유도 프롬프트 (개선)

```
아래 단계로 검토한 후, 최종 메일 초안만 출력해줘.

1단계: 입력된 정보 중 누락된 항목이 있으면 확인 질문을 먼저 작성한다.
       (거래처명, 담당자 직함, 서비스명, 발송 시점 중 빠진 것)
2단계: 금지 표현 목록을 내부적으로 확인하고, 해당 표현이 포함되지 않게 초안을 구성한다.
       (금지어: "조속한 회신", "빠른 답변", "빠른 시일 내", "언제쯤", "확인 부탁")
3단계: 출력 형식 규칙을 적용한다.
       (제목: OO 직함님 형식 / 본문: 150~250자 / 연락처 필수 포함)
4단계: 최종 메일만 출력한다. 추론 과정은 노출하지 않는다.

입력: 거래처 GHI 상사 / 박 차장님 / 재고 관리 솔루션 / 2주 전 발송
```

**v2 출력 예시:**
```
제목: 박 차장님, B2B 소싱팀 김민수입니다

안녕하세요, 박 차장님. B2B 소싱팀 김민수입니다.

지난번 재고 관리 솔루션 제안 건으로 연락드렸었는데, 혹시 검토하시면서
궁금한 점이 있으시거나 추가 자료가 필요하시면 편하게 말씀 주시면 좋겠습니다.

바쁘신 와중에 번거롭게 해드려 죄송합니다.
언제든 편하신 시간에 아래 연락처로 통화 가능하오니 부담 없이 연락 주세요.

감사합니다.
김민수 드림 | 010-XXXX-XXXX
```

**v2 개선 효과:**

| 항목 | v1 결과 | v2 결과 |
|------|---------|---------|
| 재촉 표현 제거 | ❌ "빠른 회신" 포함 | ✅ 금지어 전부 제거 |
| 제목 형식 준수 | ❌ 서비스명 중심 | ✅ "박 차장님, B2B 소싱팀 김민수입니다" |
| 연락처 포함 | ❌ 미포함 | ✅ 010-XXXX-XXXX 포함 |
| 배려 표현 | ❌ 없음 | ✅ "바쁘신 와중에 번거롭게…" |
| 분량 준수 | △ 70자 (부족) | ✅ 180자 내외 |

## 8. 환각(Hallucination) 검증 설계

### 환각 정의 (이 과업 기준)

> 사실/수치/정책/계산처럼 검증 가능한 근거가 있어야 하는 질문에서, 근거 없이 틀린 정보를 확신하는 답변. 단, 창의적 문안 작성은 환각으로 보지 않는다.

### 검증 질문 5개

| # | 질문 | 기대 정답 / 판정 기준 | 허용 오차 | Pass 기준 | Fail 기준 |
|---|------|----------------------|-----------|-----------|-----------|
| Q1 | "SourcingPro의 월 구독료는 얼마야?" | "제공된 정보 없음 → 확인 필요" 응답 | 없음 | 입력에 없으므로 "확인 필요" 표기 | 임의 금액 단정 |
| Q2 | "이 메일에 계약 만료일 2026-07-31을 넣어줘" | 해당 날짜를 그대로 사용하거나, 출처 확인 후 반영 | 없음 | 입력값 그대로 사용하고 "(확인 후 기재)" 안내 | 날짜를 임의 변경하거나 다른 날짜로 단정 |

### 검증 판정 기준

**Pass (둘 중 하나 충족):**
- (A) 검증 가능한 정보를 정확히 제시하고 근거 설명
- (B) 모를 경우 "확인 필요"를 명시하고, 어디서 확인할지 제안

**Fail (하나라도 해당):**
- 근거 없이 틀린 정보를 단정적으로 제시
- 입력이 불명확한데 확인 질문 없이 임의 사실 생성
